## 1. 서비스 개요

### Referio가 뭔가요?

**Referio는 "소개비 자동 정산 플랫폼"입니다.**

예를 들어:
- A회사(광고주)가 고객을 데려오면 수수료를 주겠다고 합니다
- 블로거 B(파트너)가 자신의 블로그에 A회사 추천 링크를 올립니다
- 독자 C가 그 링크를 통해 A회사에 문의합니다
- A회사가 C와 계약을 맺으면 → 블로거 B에게 수수료가 자동 정산됩니다

Referio는 이 과정 전체(링크 발급 → 리드 추적 → 계약 관리 → 수수료 정산)를 자동화합니다.

### 핵심 수치

| 항목 | 내용 |
|------|------|
| 서비스 URL | https://referio.kr |
| 광고주 로그인 | https://referio.kr/advertiser/login |
| 파트너 로그인 | https://referio.kr/login |
| 관리자 패널 | https://referio.kr/admin |
| 개발 환경 | http://localhost:3000 |

## 2. 등장인물

Referio에는 세 종류의 사용자가 있습니다.

### 광고주 (Advertiser)
- **누구?** 파트너 채널을 통해 고객을 유치하고 싶은 B2B 기업
- **예시:** ERP 소프트웨어 회사, HR SaaS 회사, IT 솔루션 업체
- **하는 일:**
  - 파트너를 승인/거절
  - 들어온 리드(잠재 고객)를 영업 관리
  - 계약 완료 처리 → 수수료 지급
- **로그인 방식:** 광고주 ID + 사용자 ID + 비밀번호 (3단계)

### 파트너 (Partner)
- **누구?** 추천 링크를 공유해서 수수료를 받는 개인/팀
- **예시:** 비즈니스 블로거, 유튜버, IT 인플루언서, 컨설턴트, 에이전시
- **하는 일:**
  - 광고주 프로그램에 가입 신청
  - 추천 링크를 SNS/블로그/유튜브에 공유
  - 대시보드에서 실적 확인, 수수료 수령
- **로그인 방식:** 이메일 + 비밀번호 (Supabase Auth)

### 운영자 (Admin = Referio 팀)
- **누구?** Referio 운영팀 (우리)
- **하는 일:**
  - 전체 광고주/파트너 현황 모니터링
  - 이슈 발생 시 데이터 직접 수정
  - 신규 광고주 온보딩 지원

## 3. 핵심 흐름 한눈에 보기

```
[광고주 가입] → [캠페인 설정] → [파트너 모집]
                                      ↓
[파트너 가입] → [프로그램 신청] → [광고주 승인]
                                      ↓
                          [추천 링크 발급됨]
                                      ↓
                    [파트너가 링크 SNS/블로그에 공유]
                                      ↓
                          [잠재고객이 링크 클릭]
                                      ↓
                         [문의 폼 작성 → 리드 생성]
                                      ↓
                    [광고주 대시보드에 리드 알림 도착]
                                      ↓
                    [광고주가 영업 → 계약 완료 처리]
                                      ↓
                       [수수료 자동 정산 생성됨]
                                      ↓
                    [광고주가 실제 송금 → 정산 완료]
                                      ↓
                    [파트너 대시보드에 수수료 확인]
```

## 4. 광고주 포털 운영 가이드

### 4-1. 메뉴 구성 이해

광고주 포털(https://referio.kr/advertiser)의 메뉴:

| 메뉴 | 역할 | 자주 쓰는 상황 |
|------|------|----------------|
| **대시보드** | 전체 현황 요약 | 오늘 들어온 리드 수, 이번 달 매출 확인 |
| **파트너 관리** | 파트너 승인/거절 | 신규 파트너 신청 처리 |
| **리드 관리** | 고객 파이프라인 | 영업 진행 상황 업데이트 |
| **정산 관리** | 수수료 지급 | 정산 완료 처리 |
| **캠페인** | 수수료 단가 설정 | 리드당/계약당 단가 변경 |
| **프로모션** | 이벤트 설정 | 파트너 대상 인센티브 공지 |
| **협업** | 콘텐츠 협업 제안 | 우수 파트너에게 직접 광고 제안 |
| **설정** | 계정 설정 | 비밀번호 변경, Airtable 연동 |

---

### 4-2. 파트너 관리

#### 파트너 신청이 들어왔을 때

1. **파트너 관리** 메뉴 클릭
2. **대기 중** 탭에서 신청 목록 확인
3. 파트너 프로필 클릭 → 채널 정보, 팔로워 수, 소개 확인
4. **승인** 또는 **거절** 선택

> **승인 시 자동으로 일어나는 일:**
> - 파트너에게 추천 링크가 발급됨
> - 파트너 대시보드에 해당 광고주 프로그램이 표시됨

#### 파트너 티어 (등급)

| 티어 | 의미 | 수수료 |
|------|------|--------|
| **Authorized** | 기본 등급 | 기본 단가 |
| **Silver** | 중간 등급 | 기본 단가 + 추가 |
| **Gold** | 우수 파트너 | 높은 단가 |
| **Platinum** | 최우수 | 최고 단가 |

> 티어는 광고주가 직접 변경합니다. (자동 승급은 아직 미구현)

---

### 4-3. 리드 관리 (핵심 기능)

리드 = 파트너 링크를 통해 문의한 잠재 고객

#### 계약 상태 흐름

```
pending(신규) → call_1(1차통화) → call_2(2차통화) → call_3(3차통화) → completed(계약완료)
                                                                        → invalid(무효)
                                                                        → duplicate(중복)
```

#### 각 상태 설명

| 상태 | 의미 | 언제 사용 |
|------|------|-----------|
| **pending** | 방금 들어온 신규 리드 | 초기 상태 |
| **call_1** | 1차 연락 완료 | 첫 통화/이메일 후 |
| **call_2** | 2차 연락 완료 | 추가 미팅/데모 진행 |
| **call_3** | 3차 연락 완료 | 계약 협상 단계 |
| **completed** | 계약 완료 | 실제 계약서 서명 완료 |
| **invalid** | 무효 처리 | 연락 안 됨, 조건 불일치 |
| **duplicate** | 중복 리드 | 이미 알고 있던 고객 |

#### 리드에서 할 수 있는 것들

- **메모 추가**: 통화 내용, 특이사항 기록
- **라벨 태그**: 예) #급함 #대기업 #관심있음
- **우선순위**: 낮음/보통/높음/긴급
- **담당자 지정**: 어느 영업팀원이 담당
- **유효 처리**: is_valid 체크 (유효 리드 확정)

> **중요:** `is_valid = true` 상태에서 `contract_status = 'completed'`로 변경하면 **정산이 자동 생성**됩니다.

---

### 4-4. 정산 관리

#### 정산이 자동 생성되는 조건

1. 리드가 **유효 처리(is_valid = true)** 되면 → "유효 리드 수수료" 정산 생성
2. 리드가 **계약 완료(completed)** 처리되면 → "계약 수수료" 정산 생성

#### 정산 처리 순서

1. **정산 관리** 메뉴 → **대기 중** 탭 확인
2. 파트너에게 실제 계좌이체로 송금
3. Referio에서 **완료 처리** 버튼 클릭 (또는 체크박스 선택 후 일괄 처리)
4. 파트너 대시보드에 "완료" 상태로 표시됨

#### 정산 상태

| 상태 | 의미 |
|------|------|
| **pending** | 정산 대기 중 (아직 미지급) |
| **completed** | 지급 완료 |
| **rejected** | 취소/거절 |

---

### 4-5. 캠페인 설정

캠페인 = 수수료 정책을 정의하는 설정

#### 주요 설정 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| **유효 리드 단가** | 리드 1건당 지급액 | 5만원 |
| **계약 수수료** | 계약 1건당 지급액 또는 % | 10만원 or 5% |
| **중복 체크 기간** | 동일 연락처 재유입 방지 기간 | 30일 |
| **유효 리드 기한** | 문의 후 영업 완료 목표일 | 7일 |
| **계약 기한** | 계약 완료 목표일 | 30일 |

> 캠페인 설정을 변경하면 **이후 생성되는 리드**부터 적용됩니다. 기존 리드는 영향받지 않습니다.

---

### 4-6. Airtable 연동 (Settings → 4단계 설정)

Airtable에서 리드를 자동으로 Referio로 가져올 수 있습니다.

**연동 순서:**
1. **Settings** → Airtable 탭
2. **연동 확인**: Airtable API Key 입력
3. **필드 설정**: Airtable 컬럼 ↔ Referio 필드 매핑
4. **웹사이트 설치**: Airtable Automation에 붙여넣을 코드 복사
5. **자동화 설정**: Airtable에서 Webhook 트리거 설정

## 5. 파트너 포털 운영 가이드

### 5-1. 파트너 가입 프로세스

```
1. https://referio.kr/signup 접속
2. 이메일/비밀번호 입력
3. 인증 이메일 확인
4. 프로필 설정
5. 프로그램 탐색 → 신청
6. 광고주 승인 대기
7. 승인 완료 → 추천 링크 발급
```

#### 브랜디드 가입 링크

광고주가 자신의 브랜드로 커스터마이즈된 가입 페이지를 공유할 수 있습니다.

- URL 형식: `https://referio.kr/signup/{광고주ID}`
- 예시: `https://referio.kr/signup/puzl_co_kr`

---

### 5-2. 파트너 대시보드 주요 항목

| 항목 | 설명 |
|------|------|
| **추천 URL** | 파트너가 공유할 링크 (복사 버튼 있음) |
| **실적 통계** | 클릭 수, 리드 수, 계약 수, 총 수익 |
| **티어 배지** | 현재 파트너 등급 표시 |
| **마일스톤** | 달성 목표 및 진행률 |
| **순위** | 같은 광고주 파트너 중 내 순위 |

---

### 5-3. 추천 링크 형식

파트너 추천 링크는 두 가지 형태입니다:

**형태 1 (문의 폼 직접):**
```
https://referio.kr/inquiry/{광고주ID}?ref={추천코드}
```

**형태 2 (보안 리다이렉트):**
```
https://referio.kr/security?ref={추천코드}
```

> 두 링크 모두 동일하게 작동합니다. 파트너 대시보드에서 복사한 링크를 그대로 사용하면 됩니다.

---

### 5-4. 가이드 탭 (채널별 활용법)

파트너 대시보드의 **가이드** 탭에는 채널별 활용법이 있습니다:

| 채널 | 내용 |
|------|------|
| **블로거** | 블로그 포스팅에 링크 삽입하는 방법 |
| **인스타그램** | 인스타 스토리/바이오에 링크 활용법 |
| **유튜브** | 영상 설명란 링크 삽입 방법 |
| **지인 추천** | 카카오톡/문자로 직접 공유하는 방법 |
| **카페/커뮤니티** | 온라인 커뮤니티 활용법 |

> **접근 권한:** Referio 운영팀만 사용 가능

### 6-1. 메뉴 구성

| 메뉴 | 볼 수 있는 것 |
|------|---------------|
| **대시보드** | 전체 플랫폼 현황 (광고주 수, 파트너 수, 오늘 리드 수 등) |

| **정산 관리** | 모든 정산 내역 |

| **광고주 관리** | 모든 광고주 계정 |
| **캠페인 관리** | 모든 캠페인 설정 |

### 6-2. 운영팀이 자주 하는 작업

1. 광고주가 가입 후 문의 시

1. 파트너가 "정산이 안 들어왔어요" 문의 시
2. 관리자 패널 → 정산 관리에서 해당 파트너/광고주 검색
3. settlements 테이블 상태 확인
4. 광고주에게 처리 요청하거나 직접 Supabase에서 수정

## 7. 자주 묻는 상황별 대응

### Q1. "파트너가 리드를 보냈는데 광고주 대시보드에 안 보여요"

**원인 확인 순서:**
1. 파트너의 추천 코드가 유효한지 확인 (`partner_programs` 테이블에서 해당 코드 검색)
2. 문의 폼을 올바른 광고주 URL로 접근했는지 확인
3. referrals 테이블에서 직접 데이터 존재 여부 확인

---

### Q2. "계약 완료로 바꿨는데 정산이 안 생겼어요"

**원인 및 해결:**
1. `is_valid`가 `true`인지 확인 → false면 정산 생성 안 됨
2. `partner_id`가 NULL인지 확인 → NULL이면 정산 트리거 오류
3. Supabase Dashboard → SQL Editor에서 settlements 테이블 직접 확인

> **임시 해결책:** 관리자가 Supabase에서 직접 settlements 레코드 생성

---

### Q3. "파트너가 비밀번호를 잊어버렸어요"

**파트너 (Supabase Auth 계정):**
- https://referio.kr/reset-password 에서 이메일로 재설정 가능
- 또는 Supabase Dashboard → Authentication → Users에서 직접 초기화

**광고주 (커스텀 세션 계정):**
- `/advertiser/settings`에서 본인이 변경 가능
- 잊어버린 경우 → Supabase에서 `advertisers` 또는 `advertiser_users` 테이블에서 `password_hash` 직접 수정 (bcrypt)

---

### Q4. "Airtable에서 리드가 들어오지 않아요"

**체크리스트:**
1. Airtable API Key가 올바른지 확인 (Settings에서 재입력)
2. Airtable Automation이 활성화 상태인지 확인
3. Webhook URL이 올바른지: `https://referio.kr/api/webhook/airtable`
4. 요청 헤더에 `X-API-Key`가 포함됐는지 확인
5. Vercel 로그에서 에러 메시지 확인

---

### Q5. "파트너 추천 링크가 작동하지 않아요"

**체크리스트:**
1. 링크 형식 확인: `?ref=추천코드` 파라미터가 있는지
2. `partner_programs.status`가 `approved`인지 확인
3. 해당 referral_code가 partner_programs 테이블에 존재하는지 확인

---

### Q6. "광고주가 로그인이 안 된다고 해요"

**확인 순서:**
1. 입력 형식 확인: "광고주 ID" (예: puzl_co_kr) vs "사용자 ID" (예: admin) 구분
2. `advertisers` 또는 `advertiser_users` 테이블에서 계정 존재 여부 확인
3. `advertiser_sessions` 테이블에서 만료된 세션이 있는지 확인 (세션 유효기간 7일)
4. Vercel 로그에서 로그인 에러 확인

## 8. 용어 사전

| 용어 | 설명 |
|------|------|
| **광고주 (Advertiser)** | 파트너 채널로 고객을 유치하려는 B2B 기업 |
| **파트너 (Partner)** | 추천 링크를 공유해 수수료를 받는 개인/팀 |
| **리드 (Referral/Lead)** | 파트너 링크를 통해 들어온 잠재 고객 문의 1건 |
| **추천 코드 (Referral Code)** | 파트너별 고유 식별 코드 (링크에 ?ref=코드 형태) |
| **추천 링크 (Referral URL)** | 파트너가 공유하는 고유 링크 |
| **정산 (Settlement)** | 파트너에게 지급하는 수수료 내역 |
| **캠페인** | 수수료 단가·조건을 정의하는 광고주 설정 |
| **티어 (Tier)** | 파트너 등급 (Authorized → Silver → Gold → Platinum) |
| **프로그램 (Program)** | 특정 광고주의 파트너 모집 프로그램 |
| **문의 폼 (Inquiry Form)** | 잠재 고객이 접속하는 랜딩 페이지 |
| **CRM** | Customer Relationship Management, 고객 관계 관리 도구 |
| **Airtable 연동** | Airtable의 데이터를 Referio로 자동 동기화하는 기능 |
| **협업 (Collaboration)** | 광고주가 파트너에게 콘텐츠 제작을 의뢰하는 기능 |
| **is_valid** | 리드가 유효한 잠재 고객임을 광고주가 확인한 표시 |
| **contract_status** | 리드의 현재 영업 단계 (pending~completed) |
| **광고주 ID** | 광고주 계정을 식별하는 텍스트 (예: puzl_co_kr) |
| **advertiser_uuid** | 시스템 내부에서 사용하는 광고주 고유 번호 (UUID 형식) |
| **Supabase** | Referio가 사용하는 데이터베이스 플랫폼 |
| **Vercel** | Referio가 배포되는 서버 플랫폼 |
