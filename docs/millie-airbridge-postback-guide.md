# 밀리의서재 x Referio 에어브릿지 포스트백 연동 가이드

> 밀리의서재 파트너 추천 프로그램의 전환 데이터를 에어브릿지에서 Referio로 자동 전달하기 위한 설정 가이드입니다.

---

## 1. 이 문서의 목적

에어브릿지에서 Referio로 **포스트백(Postback)**을 설정하여, 파트너가 유입시킨 사용자의 가입/구독 전환을 자동으로 추적하고 수수료 정산까지 연결하는 방법을 안내합니다.

---

## 2. 왜 필요한가

현재 밀리의서재에는 5명의 파트너(블로거, 유튜버 등)가 각자 고유한 에어브릿지 트래킹 링크를 받아 밀리의서재를 홍보하고 있습니다.

**포스트백이 없으면 어떻게 되는가:**

- 파트너가 자신의 링크로 사용자를 유입시켜도, 그 사용자가 **실제로 가입했는지, 구독까지 했는지** Referio가 알 수 없습니다.
- Referio가 모르면 → 파트너에게 수수료를 계산할 수 없습니다.
- 결과적으로 파트너는 "내가 유입시킨 고객이 몇 명인지" 확인할 수 없고, 밀리의서재도 **어떤 파트너가 얼마나 성과를 냈는지** 측정할 수 없습니다.

**포스트백을 설정하면:**

- 에어브릿지가 사용자의 가입/구독 이벤트를 **실시간으로** Referio에 전달합니다.
- Referio는 이 데이터를 바탕으로 파트너별 성과를 자동 집계하고, 수수료를 자동 정산합니다.
- 밀리의서재 담당자는 별도 수작업 없이 파트너 성과를 관리할 수 있습니다.

---

## 3. 전체 데이터 흐름

사용자가 파트너 링크를 클릭한 순간부터 수수료 정산까지의 전체 흐름입니다.

```
파트너 링크 클릭                    에어브릿지가 추적 시작
       |                                    |
       v                                    v
[사용자가 앱 설치]  ──────────>  에어브릿지 → Referio: install 이벤트
       |                         (Referio: 로그만 기록, 아직 아무 처리 없음)
       |
       v
[사용자가 회원가입]  ──────────>  에어브릿지 → Referio: sign_up 이벤트
       |                         (Referio: "이 파트너가 1명 유입시켰다" 기록 생성)
       |                         (상태: 대기중 pending)
       |
       v
[사용자가 구독 결제]  ─────────>  에어브릿지 → Referio: subscribe 이벤트
                                 (Referio: 대기중 → 완료로 전환)
                                 (DB 트리거 → 수수료 정산 자동 생성)
                                 
       ===== 끝. 수동 작업 불필요 =====
```

**핵심 포인트:** 에어브릿지가 3가지 이벤트(install, sign_up, subscribe)를 Referio에 보내주기만 하면, 나머지는 Referio가 자동으로 처리합니다.

---

## 4. 각 이벤트의 의미와 역할

| 이벤트 | 사용자 행동 | Referio 처리 | 비고 |
|--------|-----------|-------------|------|
| **install** | 앱 설치 | 로그만 저장 | 성과로 집계하지 않음. 추후 분석용 |
| **sign_up** | 회원가입 | 신규 리퍼럴(추천 기록) 생성 | 파트너가 "이 사람을 유입시켰다"는 기록 |
| **subscribe** | 유료 구독 결제 | 기존 리퍼럴을 "완료"로 전환 → 정산 자동 생성 | 실제 수수료가 발생하는 시점 |

**참고:** install 이벤트는 필수가 아닙니다. sign_up과 subscribe만 보내도 핵심 기능은 동작합니다. 다만 install도 함께 보내면 퍼널 분석에 도움이 됩니다.

---

## 5. 핵심 매핑 값: sub_id

### sub_id란?

에어브릿지 트래킹 링크에 파트너별로 심어둔 식별 코드입니다.

```
예시: 파트너 "김도서"의 트래킹 링크
https://abr.ge/millie?sub_id_1=MIL_KIM001
                       ^^^^^^^^^^^^^^^^
                       이 부분이 sub_id
```

### 왜 중요한가?

Referio는 포스트백 데이터에서 `sub_id`를 읽어 "이 이벤트가 어떤 파트너의 링크로 유입된 사용자인가"를 판별합니다.

- `sub_id`가 있으면 → 해당 파트너에게 성과 귀속
- `sub_id`가 없으면 → **어떤 파트너의 성과인지 알 수 없음** → 정산 불가

### 현재 파트너별 sub_id 할당 현황

| 파트너 | sub_id |
|--------|--------|
| 파트너 1 | `MIL_KIM001` |
| 파트너 2 | `MIL_LEE002` |
| 파트너 3 | `MIL_PARK003` |
| 파트너 4 | `MIL_CHOI004` |
| 파트너 5 | `MIL_JUNG005` |

> 위 sub_id 값들은 이미 Referio 시스템에 등록되어 있습니다. 에어브릿지 포스트백 데이터에 이 값이 포함되어야 정상 매핑됩니다.

### 1단계: 조직(Organization) 레벨 — 엔드포인트 등록

**진입 방법 (두 가지 중 편한 방법)**

- **방법 A:** 에어브릿지 대시보드 → 우측 상단 프로필 아이콘 → **조직 설정**
- **방법 B:** 대시보드 상단의 **"엔드포인트 설정하기"** 버튼 (표시되는 경우)

**설정 순서**

- [ ] 좌측 메뉴에서 **연동 관리(Integration)** 클릭

- [ ] **+ 엔드포인트 추가** 클릭

| 항목 | 입력값 |

| Method | `POST` |

- [ ] **헤더(Header) 추가** 클릭 → 아래 값 입력:

| 헤더 Key | 헤더 Value |
|----------|-----------|
| `X-API-Key` | `94818dfd1485459fbe124dbae88315f3` |

- [ ] **저장** 클릭

> 저장 후 "Referio Webhook"이 엔드포인트 목록에 나타나면 1단계 완료입니다.

---

### 2단계: 앱(밀리의서재) 레벨 — 포스트백 커넥션 추가

> **왜 필요한가:** 1단계에서 "주소"를 등록했을 뿐, 아직 실제 데이터를 보내지는 않습니다. 이 단계에서 "어떤 이벤트를, 어떤 정보와 함께 보낼 것인지"를 설정합니다.

**설정 순서**

- [ ] 에어브릿지 대시보드에서 **밀리의서재 앱** 선택
- [ ] 좌측 메뉴 → **연동(Integration)** → **광고주 포스트백(Advertiser Postback)**
- [ ] **+ 커넥션 추가(Add Connection)** 클릭
- [ ] **엔드포인트 선택:** 1단계에서 만든 **"Referio Webhook"** 선택
- [ ] **전송할 이벤트 선택** (아래 항목 체크):

| 이벤트 | 체크 여부 | 설명 |
|--------|----------|------|
| Install | 선택 (권장) | 앱 설치 추적. 필수는 아니지만 퍼널 분석에 유용 |
| Sign Up (가입) | **필수** | 회원가입 전환. 이것이 없으면 파트너 성과 기록 불가 |
| Subscribe (구독) | **필수** | 유료 구독 전환. 이것이 없으면 수수료 정산 불가 |

- [ ] **전송할 속성(Properties) 설정** — 아래 항목 반드시 포함:

| 속성 | 필수 여부 | 설명 |
|------|----------|------|
| `sub_id` (또는 `sub_id_1`) | **필수** | 파트너 식별자. 없으면 어떤 파트너 성과인지 알 수 없음 |
| User Identifier | **필수** | 사용자 고유 식별자. sign_up과 subscribe를 같은 사용자로 연결하는 데 사용 |
| Event UUID | 권장 | 이벤트 고유 ID. 중복 전송 방지에 사용 |

> **주의:** `sub_id` 속성이 체크되어 있지 않으면, 에어브릿지가 이 값을 보내지 않습니다. 그러면 Referio에서 파트너를 매칭할 수 없어 정산이 불가능합니다. 반드시 확인해 주세요.

- [ ] **저장** 클릭

## 7. 설정 완료 후 Referio에 전달할 정보

설정을 완료하셨다면, 아래 2가지를 Referio 팀에 전달해 주세요. 실제 데이터 수신 전에 **필드 구조를 검증**하기 위해 필요합니다.

### (1) Preview Payload 캡처 (필수)

> **왜 필요한가:** 에어브릿지가 보내는 JSON 데이터의 필드명이 버전이나 설정에 따라 다를 수 있습니다. Preview를 통해 Referio가 정확한 필드명을 확인하고 수신 로직을 보정합니다.

- 커넥션 설정 화면에서 **"Preview payload"** 버튼 클릭
- 표시되는 JSON 데이터 구조를 **스크린샷 캡처** 또는 **텍스트 복사**
- 특히 아래 필드가 어떤 이름으로 들어오는지 확인이 필요합니다:

```
확인이 필요한 필드:
- 이벤트 종류: event_category? eventCategory? event_type?
- 이벤트 고유 ID: event_uuid? eventUUID? event_id?
- sub_id 위치: 최상위? campaignParams 안? sub_id_1?
- 사용자 식별자: user.identifier? device.deviceUUID?
```

### (2) 이벤트 이름 확인 (필수)

> **왜 필요한가:** 에어브릿지에서 보내는 이벤트 이름 형식이 여러 가지일 수 있습니다. Referio는 아래 형식들을 모두 처리할 수 있지만, 실제로 어떤 형식으로 오는지 확인이 필요합니다.

| 이벤트 | Referio가 처리 가능한 형식 |
|--------|--------------------------|
| 앱 설치 | `install`, `airbridge.install` |
| 회원가입 | `sign_up`, `signup`, `airbridge.sign_up` |
| 구독 | `subscribe`, `airbridge.subscribe` |

- 에어브릿지 포스트백 설정 화면에서 각 이벤트의 **정확한 이름**을 확인하여 알려주세요.
- 위 목록에 없는 형식이라면 Referio 측에서 추가 대응하겠습니다.

## 8. 자주 묻는 질문 (FAQ)

### Q. 포스트백이 안 오면 어떻게 확인하나요?

Referio 측에서 수신 로그를 확인할 수 있습니다. 설정 완료 후 테스트 전환을 1건 발생시키면, Referio 팀이 데이터 수신 여부를 확인해 드립니다.

에어브릿지 대시보드의 **실시간 로그(Real-time Log)** 기능에서도 포스트백 전송 성공/실패를 확인할 수 있습니다.

### Q. sub_id를 빠뜨리면 어떻게 되나요?

해당 이벤트는 Referio에 도착하지만, **어떤 파트너의 성과인지 알 수 없어** 정산이 불가능합니다. 포스트백 속성 설정에서 `sub_id` (또는 `sub_id_1`) 항목이 반드시 체크되어 있어야 합니다.

### Q. sign_up 없이 subscribe만 오면 어떻게 되나요?

Referio는 subscribe 이벤트만 오더라도 파트너를 매칭할 수 있으면(sub_id 기반) 정산을 생성합니다. 다만, 정확한 퍼널 추적을 위해 sign_up + subscribe 모두 보내는 것을 권장합니다.

### Q. 같은 이벤트가 중복으로 전송되면 어떻게 되나요?

Referio는 이벤트 고유 ID(`event_uuid`)를 기준으로 중복을 자동 차단합니다. 같은 이벤트가 여러 번 와도 1번만 처리됩니다.

### Q. 에어브릿지에서 서버 IP를 허용해야 하나요?

Referio 서버(`referio.puzl.co.kr`)는 별도 IP 차단 없이 수신 가능합니다. 에어브릿지 발신 IP가 차단되는 경우는 없으므로 별도 설정이 필요 없습니다.

### Q. 설정 후 바로 실제 데이터가 들어오나요?

네. 포스트백 커넥션을 활성화하면, 그 시점부터 발생하는 이벤트가 실시간으로 Referio에 전달됩니다. 설정 이전의 과거 데이터는 전송되지 않습니다.

### Q. 트래킹 링크의 sub_id를 변경하고 싶으면?

Referio 팀에 연락해 주세요. Referio 시스템과 에어브릿지 트래킹 링크 양쪽 모두 변경이 필요합니다.
