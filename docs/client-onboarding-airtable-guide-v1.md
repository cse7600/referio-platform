# Referio × Airtable 클라이언트 온보딩 가이드

> **작성일:** 2026-03-27
> **대상:** Airtable을 CRM으로 사용 중인 신규 광고주
> **버전:** v1.1

---

## 이 가이드가 필요한 상황

고객사가 **자체 문의 폼**을 운영 중이고, 모든 리드(잠재 고객)를 **Airtable에서 관리**하는 경우입니다.

고객사가 별도로 설정하거나 개발할 것은 거의 없습니다.
**Referio가 주기적으로 Airtable에서 직접 데이터를 가져와 싱크합니다.**

---

## 1. 전체 흐름

```
[파트너가 Referio에서 추천 링크 발급]
            ↓
  https://고객사사이트.com/inquiry?ref=PARTNER_CODE
            ↓
  [잠재 고객이 링크 클릭 → 고객사 문의 폼 도착]
  (고객은 이름만 입력 — 추천코드 입력 불필요)
            ↓
  [폼 제출 → Airtable에 레코드 생성]
  (추천코드 필드에 PARTNER_CODE 자동 저장)
            ↓
  [Referio가 주기적으로 Airtable API 조회 → 자동 싱크]
            ↓
  [Referio가 파트너 자동 매칭 + 리드 등록]
            ↓
  [영업 진행 → Airtable에서 영업상태 변경]
            ↓
  [Referio 다음 싱크 시 상태 자동 업데이트]
            ↓
  [계약완료 상태 감지 → 파트너 수수료 자동 정산]
```

---

## 2. 고객사가 해야 할 것 (전부)

총 2가지입니다.

### ✅ 할 일 1: Referio 연동 전용 Airtable 베이스 생성 후 공유

기존 Airtable 베이스 전체를 공유하지 않고, **Referio 연동에 필요한 테이블만 담은 새 베이스를 별도로 만들어서 공유합니다.**

이 방식이 보안상 가장 안전합니다. 기존 베이스의 다른 데이터는 Referio에 노출되지 않습니다.

**생성 절차:**
1. Airtable에서 새 베이스 생성 (이름 예: `Referio 리드 연동`)
2. 아래 **3번 섹션**의 필드 구성에 맞게 테이블 설정
3. 베이스 우측 상단 **Share** → **Invite by email** → `cym@puzl.co.kr` 입력 → **Editor** 선택

> 기존 영업 CRM 베이스의 신규 리드 데이터를 이 베이스로 복사하거나 Airtable Automation으로 미러링하는 방법은 Referio 팀이 안내해드립니다.

---

### ✅ 할 일 2: 문의 폼에 추천코드 숨김 필드 추가

고객사 문의 폼에 아래 코드를 추가합니다.
고객은 아무것도 입력하지 않습니다. URL에 있는 `?ref=` 값을 자동으로 가져옵니다.

**개발팀 또는 웹 담당자에게 전달할 코드:**

```html
<!-- 폼 안에 숨겨진 필드 추가 -->
<input type="hidden" id="referral_code" name="referral_code" value="">

<script>
  (function() {
    const urlParams = new URLSearchParams(window.location.search);
    const refCode = urlParams.get('ref');
    if (refCode) {
      document.getElementById('referral_code').value = refCode;
      // 페이지 이동해도 유지되도록 세션에도 저장
      sessionStorage.setItem('referral_code', refCode);
    } else {
      const saved = sessionStorage.getItem('referral_code');
      if (saved) document.getElementById('referral_code').value = saved;
    }
  })();
</script>
```

이 값이 Airtable `추천코드` 필드에 저장되면 됩니다.

## 3. Airtable 베이스 필드 구성

### 필수 필드

| 필드명 | Airtable 타입 | 설명 |
|--------|--------------|------|
| **이름** | Single line text | 고객 이름 (식별용) |
| **추천코드** | Single line text | URL에서 자동 입력되는 파트너 코드 |
| **영업상태** | Single select | 현재 영업 단계 (아래 옵션 참고) |

> 전화번호, 이메일 등 다른 고객 정보는 있어도 되고 없어도 됩니다.
> Referio가 필요로 하는 것은 **추천코드**와 **영업상태** 두 가지뿐입니다.

---

### 영업상태 옵션 구성

아래 값 중 **유효 / 계약완료 / 무효** 세 가지는 Referio 수수료 정산에 직접 연결되므로 정확하게 만들어야 합니다.

| 옵션명 | 의미 | Referio 처리 |
|--------|------|-------------|
| `신규` | 방금 들어온 리드 (기본값) | — |
| `1차통화` | 첫 연락 완료 | — |
| `2차통화` | 추가 미팅 진행 | — |
| `3차통화` | 계약 협상 단계 | — |
| **`유효`** | 진성 고객으로 확인됨 | **수수료 지급 조건 1** |
| **`계약완료`** | 최종 계약 체결 | **수수료 지급 조건 2 (최고 단가)** |
| **`무효`** | 연락 안 됨, 조건 불일치 | 수수료 미지급 처리 |

> 옵션명은 고객사 내부 용어에 맞게 변경해도 됩니다.
> 단, 변경한 옵션명을 Referio 담당자에게 알려주어야 매핑 설정이 가능합니다.

## 4. Referio가 처리하는 것

고객사가 위 2가지를 완료하면, 이후는 Referio가 자동으로 처리합니다.

- **주기적 싱크:** Referio가 Airtable API를 주기적으로 조회해서 신규 레코드 및 상태 변경을 감지합니다
- **파트너 자동 매칭:** 추천코드 기준으로 어느 파트너가 소개한 고객인지 자동으로 연결됩니다
- **수수료 자동 정산:** 영업상태가 `유효` 또는 `계약완료`로 변경되는 순간 정산이 생성됩니다

## 6. 자주 묻는 것

**Q. 고객이 추천코드를 직접 입력해야 하나요?**
아니요. 파트너 링크에 코드가 포함되어 있어서 고객은 링크만 클릭하면 됩니다. 이름 외에 별도 입력은 없습니다.

**Q. Airtable Automation을 설정해야 하나요?**
아니요. Referio가 직접 Airtable API를 통해 데이터를 읽어오므로 Automation 설정은 필요 없습니다.

**Q. 기존 고객 데이터도 Referio로 가져올 수 있나요?**
기본적으로 신규 리드만 처리합니다. 기존 데이터 소급 처리가 필요한 경우 Referio 담당자에게 별도 문의하세요.

**Q. 영업상태 옵션명을 바꿔도 되나요?**
됩니다. 단, 변경 시 Referio 팀에 알려주어야 매핑을 업데이트할 수 있습니다.
