# 정산 관리 UX 전면 개편 + 주민번호 보안 수집 Plan

**작성일**: 2026-03-30
**상태**: 계획 수립 완료
**우선순위**: High
**레퍼런스**: https://keepermate-settlement.vercel.app/

---

## 배경 및 목적

1. 정산 처리 시 원천징수(3.3%)를 위해 파트너 주민번호 수집 필요
2. 현재 Admin/광고주 정산 페이지는 **플랫 테이블 형태** → 파트너별로 묶어 볼 수 없어 관리 불편
3. 계좌 미입력 파트너 식별 및 이메일 요청 기능이 UI만 있고 실제 동작 안 함
4. 레퍼런스 프로젝트(키퍼메이트 정산서)의 **파트너 카드 그룹핑 UI**를 Referio에 맞게 적용

---

## 핵심 보안 원칙 (SSN)

1. **서버에서만 암호화/복호화** — 평문 주민번호는 API 요청/응답에 절대 포함 안 됨
2. **AES-256-GCM 암호화** — 복호화 키는 서버 환경변수에만 존재
3. **Admin 전용 복호화** — CSV 다운로드 API에서만 복호화, 화면에는 노출 안 함
4. **표시: 입력 여부만** — 파트너/광고주/Admin 화면 모두 "입력됨/미입력" 뱃지만 표시

## UI 변경 방향 — 레퍼런스 기반

### 현재 vs 변경 후

| 항목 | 현재 | 변경 후 |
|------|------|---------|
| 정산 목록 형태 | 플랫 테이블 (행 나열) | **파트너별 카드 그룹핑** |
| 계좌 정보 위치 | 없음 | 카드 하단에 인라인 표시 |
| 주민번호 상태 | 없음 | 카드 헤더에 뱃지 표시 |
| 계좌 요청 이메일 | UI만 존재, 미작동 | 실제 발송 구현 |
| 이메일 발송 | 없음 | 파트너별 + 전체 일괄 발송 |
| CSV 다운로드 | 브라우저 클라이언트 생성 | 서버사이드 생성 (SSN 포함) |

## 화면 구성 설계

### 공통 — 파트너 카드 구조

```
┌─────────────────────────────────────────────────────────────┐
│ 홍길동   3건  합계 ₩45,000  [국민]  [대기]  [주민번호 미입력]  │
│                              [정산서 보기] [계좌요청] [이메일] ▼│
├─────────────────────────────────────────────────────────────┤
│ 정산ID   고객명    정산유형   금액       상태                  │
│ #79     김*철     유효       ₩15,000    대기                  │
│ #78     이*영     계약       ₩20,000    대기                  │
│ #77     박*수     유효       ₩10,000    대기                  │
├─────────────────────────────────────────────────────────────┤
│ 계좌정보 · 국민  ·  123-456-789012  ·  예금주 홍길동          │
└─────────────────────────────────────────────────────────────┘
```

**카드 헤더 뱃지 종류**:
- 은행명 뱃지 (계좌 입력됨): 파란색 `[국민]`
- 계좌 미입력: 빨간색 `[계좌 미입력]`
- 주민번호 입력됨: 초록색 `[주민번호 ✓]`
- 주민번호 미입력: 주황색 `[주민번호 미입력]`

**카드 헤더 버튼**:
- `정산서 보기`: 해당 파트너 정산 내역 모달 미리보기
- `계좌 등록 요청`: 계좌 미입력 시만 활성화 → 이메일 발송
- `이메일 발송`: 정산 완료 시 정산서 이메일 발송

---

### 광고주 탭 (`/advertiser/settlements`)

**범위**: 해당 광고주 프로그램에 속한 파트너만

| 항목 | 내용 |
|------|------|
| 상단 통계 카드 | 파트너 수, 총 정산 건수, 대기 금액, 이번달 정산액 |
| 파트너 카드 | 파트너별 정산 내역 그룹핑 |
| 계좌 정보 | 카드에 표시 (은행명, 계좌번호, 예금주) |
| 주민번호 | **표시 안 함** (광고주는 볼 수 없음) — 입력 여부 뱃지만 |
| 계좌 등록 요청 | 파트너별 이메일 발송 버튼 |
| 전체 발송 | "전체 이메일 발송" — 미입력 파트너 전체에게 |
| 정산 완료 처리 | 기존 기능 유지 (체크박스 선택 후 완료 처리) |
| CSV 다운로드 | 주민번호 제외 (광고주용 CSV) |

---

### Admin 탭 (`/admin/settlements`)

**범위**: 전체 광고주의 모든 파트너

| 항목 | 내용 |
|------|------|
| 상단 통계 카드 | 전체 파트너 수, 총 건수, 총 금액, 대기 건수 |
| 광고주 필터 | 드롭다운으로 광고주별 필터링 |
| 월별 필터 | 월 선택 드롭다운 |
| 파트너 카드 | 파트너별 정산 내역 그룹핑 |
| 계좌 정보 | 카드에 표시 |
| 주민번호 상태 | 입력 여부 뱃지 (값은 노출 안 함) |
| 계좌/주민번호 요청 | 파트너별 + 전체 발송 |
| CSV 다운로드 | **주민번호 포함** (서버사이드 복호화) |

## 구현 Phase

### Phase 1 — 암호화 인프라

| 작업 | 파일 | 내용 |
|------|------|------|
| 암호화 유틸 생성 | `src/lib/crypto.ts` | AES-256-GCM encrypt/decrypt |
| 환경변수 추가 | `.env.local` | `SSN_ENCRYPTION_KEY` (64자 hex) |
| Vercel 환경변수 | Vercel 대시보드 | 동일 키 등록 |

암호화 형식: `{iv_hex}:{encrypted_hex}:{authtag_hex}`

---

### Phase 2 — 파트너 프로필: 주민번호 입력

| 작업 | 파일 | 내용 |
|------|------|------|
| SSN 입력 필드 추가 | `src/app/dashboard/profile/page.tsx` | 정산정보 카드에 주민번호 입력 추가 |
| 저장 방식 변경 | 동일 | 기존 Supabase 직접 저장 → 서버 API 경유로 변경 |
| 프로필 저장 API 생성 | `src/app/api/partner/profile/route.ts` | SSN 서버 암호화 저장 |

**파트너가 보는 화면**:
- 미입력: 주민번호 입력 폼 표시
- 입력됨: `주민번호 입력 완료 ✓` 뱃지만 표시 (값 노출 없음)
- 재입력: "수정" 버튼 클릭 시 재입력 가능
