## 1. 프로젝트 개요

### 비전
B2B 서비스 기업이 콘텐츠 크리에이터, 에이전시, 지인 네트워크를 통해 잠재 고객을 유치하고, 성과에 따라 자동으로 정산까지 처리할 수 있는 어필리에이트 플랫폼.

### 미션
기업(광고주)과 추천인(파트너) 사이의 신뢰 기반 협력을 기술로 자동화한다. 추천 → 리드 접수 → 영업 파이프라인 관리 → 계약 확인 → 정산까지 전 과정을 하나의 플랫폼에서 처리한다.

### 타겟 고객
- **1차 타겟 (광고주)**: 국내 B2B SaaS, IT서비스, 기업 솔루션, 금융 서비스 등 기업 대상 서비스를 운영하는 중소기업. 영업인력이 부족하고 파트너 채널을 구축하고 싶지만 관리 도구가 없는 회사.
- **2차 타겟 (파트너)**: 비즈니스/기술 분야 블로거, 유튜버, 인스타그래머, 기업 컨설턴트, 에이전시, 지인 네트워크 보유자. B2B 서비스 추천으로 수익을 얻고 싶은 사람.

### 핵심 가치 제안
- **광고주**: 영업 직원 없이도 파트너 채널 운영. 성과 기반 정산으로 비용 효율 극대화. 실시간 리드 유입 및 CRM 파이프라인 관리.
- **파트너**: 추천 링크 하나로 수익 창출. 실시간 성과 추적. 티어 시스템으로 장기 성장 동기 부여.
- **Referio**: 플랫폼 수수료(브랜디드 콘텐츠 협업 10%), 커스텀 계약 기반 월정액 (요금제 공개 미정, 별도 문의로 운영 중).

## 3. 핵심 사용자 유형 및 시나리오

### Persona 1: 광고주 (기업 담당자)

**배경**: 중소 B2B SaaS 기업 마케팅 팀장. 신규 영업 채널이 필요하지만 전담 파트너 관리자를 두기 어렵다.

**핵심 사용 시나리오**:
1. Referio에 회원가입 → 광고주 ID 발급
2. 캠페인 설정 (유효 단가, 계약 단가 입력)
3. 파트너 신청 승인/거절
4. 파트너가 유치한 리드를 CRM처럼 관리 (통화 상태 업데이트, 메모, 라벨)
5. 계약 완료로 표시 → 정산 자동 생성 확인
6. 정산 대기 목록에서 실제 송금 후 "완료" 클릭
7. 우수 파트너에게 브랜디드 콘텐츠 협업 제안

**Pain Point 해결**:
- 파트너별 성과를 한 눈에 확인 (이달 리드/계약 수, 티어)
- 영업 파이프라인을 별도 CRM 없이 관리 가능
- 정산 계산을 수작업 없이 자동화

### Persona 2: 파트너 (콘텐츠 크리에이터/컨설턴트)

**배경**: IT 분야 블로거. 구독자 5천명. B2B 서비스 관련 콘텐츠를 쓰고 부수입을 원한다.

**핵심 사용 시나리오**:
1. Referio 파트너 가입 → 프로그램 탐색
2. 원하는 광고주의 프로그램 신청 (승인 대기)
3. 승인 후 추천 링크 복사
4. 블로그/SNS에 추천 링크 포함한 콘텐츠 게시
5. 독자가 링크 클릭 → 문의 폼 작성 → 리드 자동 기록
6. 대시보드에서 유치 고객, 유효 DB, 계약 건수, 정산 금액 확인
7. 이벤트 참여로 추가 수익 획득 가능
8. 실적 우수 시 광고주로부터 브랜디드 콘텐츠 협업 제안 수신

**Pain Point 해결**:
- 추천 성과 실시간 추적 (기존: 구두 약속만)
- 정산 내역 투명하게 확인
- 티어 시스템으로 장기 활동 동기 부여


### Persona 3: 관리자 (Referio 운영팀)

**배경**: Referio 플랫폼 운영 담당자. 전체 데이터 모니터링 및 분쟁 해결.

**핵심 사용 시나리오**:
1. 어드민 패널에서 전체 리드/정산/파트너 현황 확인
2. 이상 거래 감지 및 처리
3. 광고주 온보딩 지원

## 4. 기능 명세

### 4-1. 광고주 포털 (`/advertiser/`)

#### 인증 시스템
- **로그인 방식**: 광고주 ID + 사용자 ID + 비밀번호 (3단계, 멀티테넌트 지원)
- **세션 관리**: 커스텀 토큰 기반 (`advertiser_sessions` 테이블)
- **멀티유저**: `advertiser_users` 테이블로 동일 광고주 계정에 복수 사용자 지원 (admin/manager/viewer 역할)
- **미들웨어**: API 라우트는 미들웨어 제외, 광고주 인증은 별도 세션 방식

#### 대시보드
- 전체 파트너 수 / 활성 파트너 수
- 총 유입 고객 수 / 유효 건수
- 정산 건수 / 대기 건수
- 이번 달 정산 금액
- 최근 활동 피드 (파트너 신청, 리드 유입, 정산 생성)
- 빠른 실행 버튼 (파트너 관리, 캠페인 설정, 정산 관리)

#### 파트너 관리
- 파트너 목록 (이름, 채널, 상태, 티어, 이달 리드, 이달 계약, 추천코드)
- 승인 대기 파트너 즉시 승인/거절
- 거절된 파트너 재승인
- 이름/채널 검색 및 상태 필터 (전체/대기/승인/거절)
- 파트너 순위 표시 (승인된 파트너 기준 1~3위 메달)
- **미구현**: 파트너 초대 기능, 파트너 상세 정보 편집

#### 리드(고객) 관리 — 영업 파이프라인 CRM
- 영업 퍼널 카드 (신규 → 1차통화 → 2차통화 → 3차통화 → 계약완료, 무효/중복)
- 리드 목록 (유입일, 고객명, 연락처, 추천파트너, 계약상태, 유효여부, 메모)
- 계약 상태 인라인 변경 (드롭다운, 낙관적 업데이트)
- 유효/무효 버튼 (유효 처리 시 정산 자동 생성)
- 계약완료 처리 시 정산 자동 생성 + 토스트 알림
- 리드 상세 다이얼로그: 기본정보, 문의내용, 계약상태, 우선순위, 유효여부, 라벨, 메모
- 이름/연락처/추천코드 검색 및 상태 필터
- **미구현**: 리드 수동 추가, CSV 내보내기, 담당자 배정

#### 정산 관리
- 통계 카드 (대기 건수/금액, 완료 건수/금액, 이번달 정산, 선택된 금액)
- 대기 탭: 파트너별 정산 건 목록, 개별 완료 처리, 전체 선택 일괄 완료
- 완료 탭: 정산 이력 (정산일, 파트너, 고객, 유형, 금액, 비고)
- 정산 유형: `valid`(유효 정산), `contract`(계약 정산) 구분
- 고객명은 첫 글자만 표시 (개인정보 보호)
- **미구현**: 정산 취소, CSV 내보내기, 세금계산서 연동, 실제 송금 연동

#### 캠페인 설정
- 기본 설정: 캠페인 활성화/비활성화, 이름, 랜딩 URL, 유효 단가, 계약 단가, 수수료율, 최소 정산금액
- 티어별 단가: 티어별(authorized/silver/gold/platinum) 유효단가/계약단가 차등 설정
- 정책 설정: 중복 체크 기간(일), 유효 판정 기한(일), 계약 완료 기한(일)
- 프로모션 탭: 진행 중인 프로모션 조회
- **미구현**: 신규 캠페인 생성 버튼 (현재 "준비 중"), 캠페인 복수 운영, 캠페인 삭제

#### 이벤트/프로모션 관리
- 이벤트 생성: 제목, 유형(이벤트/보너스/랭킹), 리워드 설명, 기간, 상세 내용, 파트너 공개 여부
- 이벤트 목록: 제목, 유형, 상태(진행중/종료/임시저장), 비공개 여부 표시
- 이벤트 종료/재시작
- 파트너 대시보드에 배너로 자동 노출 (is_visible_to_partners = true)
- **미구현**: 이벤트 수정, 이벤트 삭제

#### 브랜디드 콘텐츠 협업
- 파트너 탐색 탭: 계약 실적 상위 파트너 목록, 협업 제안 버튼
- 협업 제안: 파트너 선택, 제목, 콘텐츠 유형(블로그/유튜브/인스타/틱톡/기타), 예산(플랫폼 수수료 10% 자동 계산), 마감일, 브리프
- 협업 관리 탭: 진행 중인 협업 목록 (제목, 상태, 파트너, 금액, 마감일)
- 협업 상세: 기본정보, 결과물 URL 확인, 검수완료/수정요청 액션, 취소, 메시지 채팅
- 협업 상태 흐름: requested → accepted → in_progress → submitted → revision/completed → paid
- **미구현**: 자동 정산 연동 (completed/paid 상태 변경 시 정산 생성)

### 4-3. 추천 링크 문의 폼 (`/inquiry/[advertiserId]`)

이 페이지는 파트너가 고객에게 공유하는 핵심 랜딩 페이지다.

- URL 형태: `https://도메인/inquiry/광고주ID?ref=파트너추천코드`
- 입력 필드: 이름(필수), 연락처(필수), 문의내용(선택)
- 제출 시 `?ref` 파라미터의 추천 코드를 함께 전송
- 접수 성공 시 완료 메시지 표시
- **미구현**: 광고주 브랜딩 (로고, 색상 적용), 광고주 서비스 소개 내용, 이메일 필드

**주의**: 현재 파트너 대시보드 코드에서 추천 URL은 `https://referio.kr/security?ref=코드` 형태로 생성되고 있으나, 실제 문의 폼은 `/inquiry/[advertiserId]?ref=코드` 형태다. 두 URL이 다른 것은 확인된 사실이며, `/security` 페이지가 실제 리다이렉트 역할을 하는지 여부는 별도 확인 필요.

### 4-4. 관리자 패널 (`/admin`)

### 핵심 엔티티와 관계

```
[Referio 플랫폼]
       │
  ┌────┴────┐
  │         │
광고주    파트너
(advertisers)  (partners)
  │         │
  └────┬────┘
       │
  [partner_programs]
  광고주-파트너 연결
  (referral_code, tier, status)
       │
  [referrals]
  파트너가 유치한 리드
       │
  [settlements]
  정산 내역
```

### 주요 테이블

| 테이블 | 설명 | 핵심 컬럼 |
|--------|------|-----------|
| `advertisers` | 광고주 회사 계정 | advertiser_id(text), company_name, plan_id, status |
| `advertiser_users` | 광고주 멀티유저 | advertiser_id, user_id, role(admin/manager/viewer) |
| `advertiser_sessions` | 광고주 세션 | token, advertiser_id, user_id |
| `advertiser_plans` | 요금제 | name, max_partners, max_programs, features(JSON) |
| `partners` | 파트너 개인 계정 | auth_user_id, name, tier, status, referral_url, referral_code |
| `partner_programs` | 파트너-광고주 프로그램 참가 | partner_id, advertiser_id, referral_code, tier, status |
| `referrals` | 파트너가 유치한 리드 | partner_id, advertiser_id, name, phone, contract_status, is_valid, commission_amount |
| `settlements` | 정산 내역 | partner_id, advertiser_id, referral_id, type(valid/contract), amount, status |
| `partner_stats` | 파트너 집계 통계 (캐시) | partner_id, total_referrals, total_valid, total_contracts, total_settlement |
| `partner_promotions` | 광고주가 만든 이벤트/프로모션 | advertiser_id, title, promotion_type, is_visible_to_partners, status |

### referrals.contract_status 값
`pending` → `call_1` → `call_2` → `call_3` → `completed` (또는 `invalid` / `duplicate`)

### collaborations 관련 테이블
- `content_type`: blog / youtube / instagram / tiktok / other
- `status`: requested → accepted → in_progress → submitted → revision → completed → paid (또는 declined / cancelled)

## 6. 비즈니스 로직

### 6-1. 리드 유입 → 정산 플로우

```
파트너가 추천 링크 공유
        ↓
잠재 고객이 /inquiry/[advertiserId]?ref=코드 접속
        ↓
이름, 연락처, 문의내용 입력 → 제출
        ↓
/api/inquiry POST 호출
- referral_code로 파트너 식별
- referrals 테이블에 리드 생성 (contract_status: 'pending')
        ↓
광고주 리드 관리 화면에 즉시 표시
        ↓
광고주가 리드를 영업 파이프라인으로 관리
(pending → call_1 → call_2 → call_3 → completed)
        ↓
[유효 처리 시]
광고주가 is_valid = true 설정
→ settlements 자동 생성 (type: 'valid', amount: valid_amount)

[계약 완료 처리 시]
광고주가 contract_status = 'completed' 설정
→ settlements 자동 생성 (type: 'contract', amount: contract_amount)
→ 파트너 정산 대기 목록에 표시

[정산 집행 시]
광고주가 실제 송금 후 settlements 상태를 'completed'로 변경
→ 파트너 정산 현황에 완료 표시
```

### 6-2. 멀티테넌트 구조

Referio는 하나의 플랫폼에 여러 광고주가 독립적으로 존재하는 멀티테넌트 구조다.

- 각 광고주는 고유 `advertiser_id`(텍스트)로 식별됨
- 파트너는 복수의 광고주 프로그램에 동시 참가 가능
- `partner_programs` 테이블이 N:M 관계 중개
- 파트너 추천 코드는 프로그램별로 다르게 발급
- 리드/정산 모두 `advertiser_id` 컬럼으로 격리

**광고주 인증과 파트너 인증이 완전히 분리**:
- 광고주: 커스텀 세션 토큰 방식 (Supabase Auth 미사용)
- 파트너: Supabase Auth (이메일/비밀번호)

### 6-3. 티어 시스템

파트너 티어: Authorized → Silver → Gold → Platinum

- 티어별 단가 차등 적용 가능 (광고주가 캠페인에서 설정)
- 티어 업그레이드 조건: `min_contracts` (최소 계약 수) 기반
- 현재 티어 자동 승급 로직은 UI에서 확인되지 않음 (수동 관리 가능성)

### 6-4. 브랜디드 콘텐츠 협업 수수료

- 예산의 10%를 Referio 플랫폼 수수료로 자동 계산
- 파트너 수취분 = 예산 - 플랫폼 수수료
- 계산은 클라이언트에서 실시간 표시 (`budgetNum * 0.1`)

## 8. 기술 부채 및 리스크

### 즉각 조치 필요

| 항목 | 심각도 | 설명 |
|------|--------|------|
| 테스트 계정 하드코딩 | 높음 | `/login/page.tsx`에 계정 정보 노출. 서비스 오픈 전 반드시 제거 |
| 추천 URL 불일치 | 높음 | 파트너 대시보드가 생성하는 URL(`/security?ref=`)과 실제 문의 폼 경로(`/inquiry/[id]?ref=`)가 다름. 파트너가 잘못된 링크를 공유할 경우 리드 유실 |
| 관리자 패널 인증 확인 | 높음 | `/admin` 경로 접근 제어 미확인 |

### 중기 리스크

| 항목 | 심각도 | 설명 |
|------|--------|------|
| 광고주 인증 방식 | 중간 | Supabase Auth를 사용하지 않는 커스텀 세션 방식. 보안 감사 및 토큰 만료 정책 확인 필요 |
| 정산 자동 생성 중복 방지 | 중간 | 동일 referral에 대해 is_valid와 contract_status 변경이 동시에 일어날 경우 중복 정산 생성 가능성 검토 필요 |
| partner_stats 테이블 동기화 | 낮음 | 집계 캐시 테이블. 원본 데이터(referrals, settlements)와 불일치 발생 시 파트너가 잘못된 통계 확인 |
| 캠페인 1개 제한 | 낮음 | 현재 UI가 광고주당 캠페인 1개를 전제로 설계됨. 멀티 캠페인 지원 시 구조 변경 필요 |

### 기술 스택 관찰 사항

| 항목 | 설명 |
|------|------|
| `_archived` 폴더 존재 | `src/app/api/advertiser/_archived/`에 미사용 API 존재 (campaigns, collaborations, credits). 불필요하면 삭제, 재활용 계획이 있으면 명시 |
| 미들웨어 범위 | 현재 미들웨어는 파트너 인증(Supabase Auth)만 처리. 광고주 인증은 각 API 라우트에서 개별 처리 |
| 이메일 알림 | 리드 유입, 정산 생성 등 주요 이벤트에 대한 이메일 알림 없음 |
