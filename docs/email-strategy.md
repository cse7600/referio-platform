## 2. 현재 구현된 이메일 (3종)

### [P1] 파트너 가입 환영 이메일

| 항목 | 내용 |
|------|------|
| **트리거** | 파트너 최초 소셜 로그인 (OAuth 가입 완료 시) |
| **수신자** | 신규 파트너 |
| **발송 코드** | `sendWelcomeEmail()` — `src/app/auth/callback/route.ts` |
| **제목** | `[Referio] 가입을 환영합니다!` |
| **본문 내용** | 환영 메시지, 3단계 시작 가이드, "프로그램 둘러보기" 버튼 |
| **현재 상태** | ✅ 운영 중 |
| **개선 포인트** | 파트너 이름 개인화, 추천 가능 프로그램 3개 노출, 온보딩 체크리스트 추가 |

---

### [P2] 파트너 승인 알림 이메일

| 항목 | 내용 |
|------|------|
| **트리거** | 광고주가 파트너 신청을 "승인" 클릭 |
| **수신자** | 승인된 파트너 |
| **발송 코드** | `sendPartnerApprovalEmail()` — `src/app/api/advertiser/partners/[id]/route.ts` |
| **제목** | `[Referio] {프로그램명} 파트너로 승인되었습니다` |
| **본문 내용** | 추천 코드 (강조 표시), 추천 링크, "대시보드로 이동" 버튼 |
| **현재 상태** | ✅ 운영 중 |
| **개선 포인트** | 커미션 구조 명시 (유효 단가 / 계약 단가), 첫 추천 행동 유도 CTA, 예상 수익 계산기 링크 |

---

### [A1] 광고주 새 리드 접수 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 문의폼 제출 또는 Airtable 웹훅으로 신규 리드 생성 |
| **수신자** | 광고주 담당자 (`contact_email`) |
| **발송 코드** | `sendAdvertiserNewLeadEmail()` — `src/app/api/inquiry/route.ts`, `src/app/api/webhook/airtable/route.ts` |
| **제목** | `[Referio] 새 리드 접수 — {리드명}` |
| **본문 내용** | 리드 이름, 연락처, 유입경로 (문의폼/Airtable), 파트너 귀속 여부, "리드 관리 바로가기" 버튼 |
| **현재 상태** | ✅ 운영 중 |
| **개선 포인트** | 누적 리드 수 표시, 이번 달 목표 대비 달성률, 담당자 배정 바로가기 딥링크 |

## 3. 구현 예정 이메일 (5종)

> 우선순위 순서로 정렬. 구현 확정 시 담당자가 체크 후 개발 요청.

### [P3] 파트너 리드 유효 확정 알림 ⭐ 최우선

| 항목 | 내용 |
|------|------|
| **트리거** | 광고주가 리드를 "유효" 처리 (또는 Airtable 웹훅에서 유효 상태 수신) |
| **수신자** | 해당 리드를 추천한 파트너 |
| **제목 (안)** | `[Referio] {리드명}님이 유효 고객으로 확정됐습니다 🎉` |
| **본문 내용 (안)** | 고객명, 광고주명, 유효 커미션 금액, "내 실적 보기" 버튼 |
| **비즈니스 가치** | 파트너 동기부여 핵심 순간 — 이 알림이 추가 추천 활동 유도 |
| **구현 상태** | ❌ 미구현 |
| **구현 위치** | `src/app/api/advertiser/referrals/[id]/route.ts` (유효 처리 API) |

---

### [A2] 광고주 신규 파트너 신청 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 파트너가 광고주 프로그램에 신청 (pending 상태 생성) |
| **수신자** | 광고주 담당자 (`contact_email`) |
| **제목 (안)** | `[Referio] 새 파트너가 프로그램을 신청했습니다` |
| **본문 내용 (안)** | 파트너 이름, 파트너 소개 (있으면), "승인하러 가기" 버튼 |
| **비즈니스 가치** | 승인 지연 방지 → 파트너 이탈 방지 (현재 승인 안 하면 파트너가 까먹음) |
| **구현 상태** | ❌ 미구현 |
| **구현 위치** | `src/app/api/advertiser/partners/route.ts` (신청 API) |

---

### [P4] 파트너 계약 완료 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 광고주가 리드를 "계약 완료" 처리 |
| **수신자** | 해당 리드를 추천한 파트너 |

| **본문 내용 (안)** | 고객명, 계약 커미션 금액, 누적 커미션 합계, "정산 현황 보기" 버튼 |
| **비즈니스 가치** | 파트너 신뢰 구축 — 커미션이 실제로 쌓이고 있다는 체감 |
| **구현 상태** | ❌ 미구현 |
| **구현 위치** | `src/app/api/advertiser/referrals/[id]/route.ts` (계약완료 처리 API) |

### [P5] 파트너 정산 처리 알림

| 항목 | 내용 |

| **구현 상태** | ❌ 미구현 |

### [P6] 파트너 비활성 리마인더 (월 1회 배치)

| 항목 | 내용 |
|------|------|

| **수신자** | 비활성 파트너 |
| **제목 (안)** | `[Referio] 최근 추천이 없으시네요 — 이번 달 수익 기회를 잡아보세요` |
| **본문 내용 (안)** | 참여 중인 프로그램 목록, 이번 달 인기 프로그램, "지금 공유하기" 버튼 |
| **비즈니스 가치** | 휴면 파트너 재활성화 — 저비용 고효율 리텐션 채널 |
| **구현 상태** | ❌ 미구현 |
| **구현 위치** | Vercel Cron Job 또는 Supabase pg_cron |

## 4. 전체 트리거 맵

```
파트너 여정
────────────────────────────────────────────────────
가입          → [P1] 환영 이메일 ✅
프로그램 승인  → [P2] 승인 알림 ✅
리드 유효     → [P3] 유효 확정 알림 ❌ (구현 예정)
리드 계약     → [P4] 계약 완료 알림 ❌ (구현 예정)
정산 처리     → [P5] 정산 알림 ❌ (구현 예정)
30일 비활성   → [P6] 리마인더 ❌ (구현 예정)

광고주 여정
────────────────────────────────────────────────────
신규 리드 접수 → [A1] 리드 알림 ✅
파트너 신청   → [A2] 파트너 신청 알림 ❌ (구현 예정)
```

## 5. 콘텐츠 가이드라인

### 브랜드 보이스
- 간결하고 명확하게. 불필요한 수식어 없음
- 수치 강조 (금액, 건수, 달성률)
- 1개의 이메일 = 1개의 핵심 메시지 + 1개의 CTA 버튼

### 제목 작성 원칙
- 이모지 1개 허용 (끝에 배치)
- 구체적 수치 포함 시 오픈율 상승
- 예: `[Referio] 홍길동님이 계약했습니다! 커미션이 확정됐습니다 💰`

### 발신자 표시
- 발신자 이름: `Referio`
- 발신 주소: `noreply@updates.puzl.co.kr`
- 수신 거부 링크: 프로필 설정 페이지로 연결 (현재 미구현)

## 6. 성과 측정 계획

> Resend 대시보드에서 확인 가능한 지표

| 지표 | 목표 | 비고 |
|------|------|------|
| 발송 성공률 | 99% 이상 | Resend 내장 |
| 오픈율 | 40% 이상 | 거래형 이메일 기준 |
| 클릭률 | 15% 이상 | CTA 버튼 클릭 |
| 이탈율 (Bounce) | 2% 이하 | 도메인 평판 관리 |
