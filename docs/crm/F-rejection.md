### 발송 타이밍 / 조건

| 항목 | 내용 |
|------|------|
| **발송 시점** | 광고주가 거절 버튼 클릭 즉시 (API 응답 내 또는 background job) |
| **최대 지연** | 30초 이내 (파트너 신뢰를 위해 즉각 알림) |
| **발송 조건** | `partner_programs.status` 가 `'rejected'`로 전환되는 시점 |
| **중복 방지** | 동일 `partner_program_id` + `event_type='rejection'`으로 발송 로그 체크 |
| **재발송 불가** | 같은 신청 건에 대해 1회만 발송 |

### 구현 노트

#### API 트리거 포인트

```
PATCH /api/advertiser/partner-programs/[id]
  body: { status: 'rejected', rejection_reason: '...' }
  → status 변경 완료 후 sendRejectionEmail() 호출
```

#### 분기 로직 (이메일 렌더링)

```
if (rejection_reason) {
  // 거절사유 있는 버전: "광고주가 전달한 내용" 블록 렌더링
} else {
  // 거절사유 없는 버전: "별도 사유를 남기지 않았습니다" 블록 렌더링
}

if (alternativePrograms.length > 0) {
  // 최대 3개 프로그램 목록 렌더링
  // Subject: B안 사용
  // Preheader: "다른 기회가 있습니다" 버전
} else {
  // "마켓플레이스 확인" 단순 링크
  // Subject: A안 사용
  // Preheader: "언제든 다시 도전" 버전
}
```

#### 대안 프로그램 조회 쿼리 기준

- `status = 'active'`인 프로그램 중 해당 파트너가 **아직 신청하지 않은** 프로그램
- 해당 파트너가 **이미 승인된** 프로그램 제외
- 최신 생성 순 정렬, 최대 3개
- 조회 시점: 거절 이벤트 발생 시

#### 함수 시그니처 (email.ts 추가 대상)

```typescript
export async function sendPartnerRejectionEmail(params: {
  partnerEmail: string;

  rejectionReason?: string;
  alternativePrograms?: Array<{
    name: string;
    advertiserName: string;
    commissionDescription: string;
  }>;
}): Promise<void>
```

### 디자인 가이드

| 요소 | 지침 |
|------|------|
| **톤** | 중립적이고 따뜻함. 비난 없음. 미래지향적 |
| **거절 사실 위치** | 상단 1/3 이내에서 명확하게 — 숨기거나 돌려말하지 않음 |
| **거절사유 표시** | 인용구(blockquote) 스타일로 광고주 원문 그대로. 편집 없음 |
| **CTA 색상** | 브랜드 기본색 (다른 긍정 이메일과 동일) — 거절이라고 어둡게 처리 금지 |
| **하단 대안 섹션** | 배경색 구분 (밝은 회색 또는 브랜드 라이트 컬러)으로 "새로운 기회" 느낌 부각 |
