### 발송 조건 메모

- **조회 쿼리**: `partner_programs` 테이블에서 `approved_at < now() - interval '7 days'` AND `approved_at > now() - interval '8 days'` AND 해당 파트너의 `referrals` 건수 0
- **프로그램 목록 동적 삽입**: 해당 파트너가 승인된 모든 프로그램을 순서대로 나열. 각 항목에 프로그램명 + 추천링크 + 커미션 요약
- **중복 발송 방지**: 이미 이 이메일을 받은 파트너에게 재발송하지 않음 (발송 로그 테이블 활용)
- **활동 감지 기준**: `referrals` 테이블에 해당 파트너 기준 레코드 1건 이상이면 발송 제외
- **구현 위치**: `/api/cron/nudge-no-activity` (신규 Cron 엔드포인트)
- **Vercel Cron**: `vercel.json`에 매일 10:00 KST 실행 스케줄 등록

### 발송 조건 메모

- **트리거**: 어드민 또는 광고주가 신규 프로그램을 `is_active=true`로 전환하는 시점. `/api/admin/programs/[id]/activate` 또는 광고주 프로그램 활성화 API에서 이메일 발송 호출
- **수신자 필터링**: `partner_programs` 테이블에서 해당 파트너의 활성 참가 프로그램 수를 집계. `count <= 2`인 파트너만 발송 대상. 기준값은 상수로 관리 (향후 조정 가능)
- **이미 신청한 파트너 제외**: 신규 프로그램에 이미 `partner_programs` 레코드가 있으면 발송 제외
- **신청 링크**: `https://referio.kr/dashboard/marketplace/[programId]` (직접 해당 프로그램 상세로 이동)
- **대량 발송 고려**: 파트너 수가 많을 경우 큐 기반 처리 (Vercel Edge Queue 또는 배치 분할)
- **프로그램_설명_1줄**: `programs` 테이블의 `description` 필드 앞 50자 또는 별도 `short_description` 컬럼 활용

- **최초 발송**: 승인일 기준 30일 시점 파트너를 일 1회 배치로 감지하여 발송. 집계 기준은 해당 파트너의 `approved_at`부터 발송 시점까지 전체 기간
- **정기 발송**: 매월 1일 전월(1일 00:00 ~ 말일 23:59 KST) 기준 집계. 해당 기간 내 `approved_at`이 있는 모든 파트너 대상
- **집계 쿼리**:
  - 리드 수: `referrals` 테이블에서 해당 파트너 + 기간 필터
  - 커미션 합계: `settlements` 테이블에서 해당 파트너 + 기간 필터 + `status != 'cancelled'`
  - 전환율: 리드 수 대비 `is_valid=true` 건수 비율
- **프로그램별 KPI 동적 삽입**: 파트너가 참가 중인 프로그램 각각에 대해 위 집계를 분리 실행하여 테이블 형식으로 조합
- **성과 유무 분기**: `전체_리드수 >= 1` 또는 `커미션_총액 > 0`이면 "성과 있음" 버전. 둘 다 0이면 "성과 없음" 버전. 템플릿 엔진(Handlebars 등) 또는 서버 사이드 분기로 처리
- **증감율 계산**: `(이번달 - 지난달) / 지난달 * 100`. 지난달이 0이면 "지난달 데이터 없음"으로 표기
- **구현 위치**: `/api/cron/monthly-performance-report` (신규 Cron 엔드포인트)
- **Vercel Cron**: `vercel.json`에 `0 0 1 * *` (매월 1일 00:00 UTC = 09:00 KST) 등록
- **이메일 크기 고려**: 프로그램이 많은 파트너는 프로그램별 KPI 섹션이 길어질 수 있음. 최대 5개 프로그램까지만 본문에 포함, 나머지는 "대시보드에서 전체 확인" 링크로 안내
